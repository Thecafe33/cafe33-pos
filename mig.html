🔧 Migrate bedBookings_archive

Chuẩn hóa dữ liệu Firestore về đúng cấu trúc { bookings: {...}, archivedAt: "..." } theo từng ngày YYYY-MM-DD

Document ID: YYYY-MM-DD (ví dụ: 2026-03-10)
Fields:
  archivedAt: string ISO — thời điểm archive
  bookings: object — map các booking theo key push ID
    └ mỗi booking có: phone, name, date, start, end, status, bedId, dateKey, ...

Các trường hợp lỗi script sẽ sửa:
  ① Document không có field bookings (data nằm thẳng ở root) → bọc vào bookings
  ② Document ID không đúng dạng YYYY-MM-DD → tạo lại doc đúng, xóa doc sai
  ③ Field archivedAt bị thiếu → tự thêm giá trị mặc định
  ④ dateKey trong mỗi booking bị sai hoặc thiếu → cập nhật theo doc ID
